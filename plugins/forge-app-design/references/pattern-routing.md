# Pattern Routing

Use page role to select the pattern before implementation.

| Page Role | Preferred Pattern | Required First-Viewport Evidence |
|---|---|---|
| dashboard | dashboard-control-tower-protask | trends, risks, priority tasks, activity, entry points |
| list | rich-entity-list-protask | filters, compact summary, table header, first rows |
| detail | split-pane-triage-protask | evidence, status, root cause, impact, next action |
| action | action-form-protask | grouped form, right context, save feedback |
| workflow | kanban-workflow-protask | lanes, counts, actionable cards, diagnostic links |

Do not force every page through one generic layout template.
